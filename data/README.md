# Data Sources

The standalone pipeline is designed to run from files contained in this
repository plus the trained model artifact downloaded from GitHub Releases.

This document explains the role and provenance of the data files. For the Colab
execution workflow, see `../COLAB_COMPATIBILITY.md`.

## Committed Data Files

### `data/raw/df3_availability.csv`

Locked development/training snapshot used for model development, model
selection, and the frozen-test contract.

This file must remain unchanged. Do not replace it with newer MTA data or with
a processed Session 6 output.

Required columns:

- `Month`
- `Equipment Code`
- `Total Outages`
- `Scheduled Outages`
- `Unscheduled Outages`
- `Entrapments`
- `Time Since Major Improvement`
- `AM Peak Availability`
- `PM Peak Availability`
- `24-Hour Availability`

Strongly preferred contextual columns:

- `Equipment Type`
- `Borough`
- `Station Name`
- `Station MRN`
- `Station Complex Name`
- `Station Complex MRN`

### `data/raw/df1_station_master.csv`

Static station metadata / lookup table with GTFS station identifiers, ADA
fields, borough, line, latitude, longitude, and georeference. This file is
committed because it is small and useful context, but it is not required by the
locked Session 6 / Session 7 Colab workflow.

### `data/external/MTA_NYCT_Subway_Elevator_and_Escalator_Availability__Beginning_2015.csv`

Committed reference MTA export used by Session 6 and Session 7.

The included file was downloaded on June 16, 2026. It contains 80,316 rows of
monthly elevator/escalator availability data from January 2015 through April 2026.

Session 6 converts this official export into canonical pipeline format and uses
target-known later prediction months for prospective external evaluation. Session 7 uses
the latest available feature month from this export to generate unlabeled
next-month risk scores.

## Generated Data Files

### `data/processed/session6_latest_canonical_availability.csv`

Generated by Session 6 from the external MTA export.

This file is ignored by Git because it is reproducible. Do not copy it over
`data/raw/df3_availability.csv`.

### `outputs/predictions/latest_unlabeled_risk_scores.csv`

Generated by Session 7. With the included June 16, 2026 export, Session 7 uses
April 2026 feature rows to generate unlabeled May 2026 risk scores.

## Known Source And Provenance

- Source organization: Metropolitan Transportation Authority via data.ny.gov.
- Source page:
  <https://data.ny.gov/Transportation/MTA-NYCT-Subway-Elevator-and-Escalator-Availabilit/rc78-7x78/about_data>
- Historical July 2025 source export:
  `data/raw/MTA_NYCT_Subway_Elevator_and_Escalator_Availability_accessed_July_2025 copy.csv`
  was used to reconstruct the locked `df3_availability.csv` snapshot. That
  original copy CSV is not committed.
- Current committed reference external export:
  `data/external/MTA_NYCT_Subway_Elevator_and_Escalator_Availability__Beginning_2015.csv`.
- The current `df3_availability.csv` is an exact canonical subset of the July
  2025 official export.
- Reconstructed Session 5.5 lineage: keep the official 22 columns unchanged,
  filter to `Equipment Type == "Elevator"`, filter to
  `Month >= 2021-01-01`, and sort by month descending while preserving source
  order within each month.
- No aggregation, column renaming, availability scale conversion, or
  missing-value imputation is required to regenerate `df3_availability.csv`.
- The June 16, 2026 official export stores availability fields as percent
  strings such as `100%`; the canonical converter normalizes those fields to
  proportions in `[0, 1]` for pipeline compatibility.
- `Time Since Major Improvement` is measured in months.
- Unscheduled outage counts represent distinct incidents.
- Newly introduced elevators remain eligible once they satisfy the required
history window.

Use `src/mta_elevator_pipeline/source_to_canonical.py` to create regenerated or
prospective canonical files at separate paths. It refuses to overwrite
`data/raw/df3_availability.csv`.

## Key Field Definitions

Official field definitions are available on the MTA/data.ny.gov source page.
The modeling-relevant fields used in this project are interpreted as follows:

- `Month`: the calendar month summarized by the row.
- `Equipment Code`: the elevator or escalator asset identifier.
- `Total Outages`: the total number of equipment outages.
- `Scheduled Outages`: the number of outages where an elevator or escalator is
  taken out of service for scheduled maintenance.
- `Unscheduled Outages`: the number of unplanned or unexpected interruptions of
  elevator or escalator service.
- `Entrapments`: the number of entrapments.
- `Time Since Major Improvement`: months since the last major improvement on
  the asset.
- `AM Peak Availability`: the percent of the time that the elevator or
  escalator was available during the AM peak period, 6 a.m. to 10 a.m., per
  month.
- `PM Peak Availability`: the percent of the time that the elevator or
  escalator was available during the PM peak period, 3 p.m. to 7 p.m., per
  month.
- `24-Hour Availability`: the percent of the time that the elevator or
  escalator was available, per month.

## Using Newer MTA Data

To run Session 6 and Session 7 with a newer official MTA export:

1. Download the CSV from the MTA/data.ny.gov source page.
2. Rename the downloaded file to remove the appended download date so the path is exactly:
   `data/external/MTA_NYCT_Subway_Elevator_and_Escalator_Availability__Beginning_2015.csv`
3. Replace the existing file at that path.
4. Rerun Session 6 and Session 7.

The pipeline currently expects that exact external CSV path.

Do not use newer data to change the locked model, threshold, target definition,
feature set, or frozen-test result. Newer data is for prospective evaluation and
latest-score generation unless a separate retraining workflow is explicitly
created and documented.

## Data Limitations And Assumptions

Each row is treated as one equipment asset summarized for one complete calendar
month. The pipeline assumes the MTA monthly export preserves that structure.

Users replacing the external export with newer MTA data should verify that the
downloaded file preserves the expected column names and monthly structure before
rerunning Session 6 or Session 7.

No separate external `df2` source is required for this repository.

When replacing the external export, optionally update
`data/provenance-template.yaml` or otherwise record the download date, source
URL, filename, row count, month range, and any relevant notes.

### X-Suffixed Equipment Codes

The source data includes 50 X-suffixed elevator equipment codes. In the reviewed
exports these rows have zero `Unscheduled Outages`, zero `Entrapments`, and
missing `Time Since Major Improvement`.

These assets are most likely elevators whose maintenance is handled by an
outside contractor rather than directly by MTA, which may explain the missing
maintenance-history fields. This interpretation has not been formally confirmed
from MTA documentation.

The locked supervised modeling pipeline excludes X-suffixed equipment from
model eligibility because their outcome/history fields appear structurally
different from the rest of the elevator fleet.

## Development Versus New Data

Use `data/raw/df3_availability.csv` for the locked development and frozen-test
contract. Do not switch development inputs to newly downloaded data midway,
because changing the dataset would invalidate comparisons and the frozen-test
contract.

With the included example data:

- Earlier eligible months through January 1, 2025 were used for development and
  model selection.
- February 1, 2025 through April 1, 2025 were reserved for the one-time frozen
  final test.
- Session 6 evaluates later target-known prediction months from June 1, 2025
  through March 1, 2026.
- Session 7 uses April 1, 2026 feature rows to generate unlabeled risk scores
  for the next target month, May 1, 2026.

## Useful Optional Sources

These are not required for the first prediction pipeline but could improve
interpretation or future modeling:

- Equipment installation and major-improvement dates.
- Maintenance ownership or contractor records.
- Elevator manufacturer and model.
- Inspection records and defects.
- Work-order or repair-duration records.
- Ridership or estimated elevator usage.
- Weather data by month.
- Planned capital-work schedules.
- Station accessibility and transfer-complex metadata.

## Not Required

- Station coordinates.
- Subway-line geometries.
- Cluster assignments.
- Map or frontend files.
